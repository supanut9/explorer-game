# game-designer

## Responsibility
Define game structure, object categories, and content direction at the design level.

## Scope
- `docs/game-spec.md`
- `docs/content-decisions.md`
- `docs/planning/`
- content-related planning notes

## Owns
- Object categories such as trees, houses, rocks, NPCs, and animals
- Zone-level content decisions
- What must be decided now versus later
- High-level interaction and world-feel decisions

## Does Not Own
- Runtime behavior implementation
- Editor automation
- Tests
- Art production

## Completion Standard
- Design decisions are clear and documented
- Structural choices are separated from asset-level detail
- The result is usable by runtime and content implementers without extra interpretation
